# 에러 바운더리 테스트 가이드

## 개요

이 프로젝트에는 두 가지 전역 에러 바운더리가 구현되어 있습니다:

1. **`app/error.tsx`** - 일반 에러 바운더리 (레이아웃 내 에러 처리)
2. **`app/global-error.tsx`** - 루트 레이아웃 에러 바운더리 (html, body 태그 포함)

## 테스트 방법

### 1. 테스트 페이지 사용 (권장)

브라우저에서 `/test-error` 페이지로 이동하여 다양한 에러 시나리오를 테스트할 수 있습니다.

```bash
# 개발 서버 실행 후
http://localhost:3000/test-error
```

**테스트 항목:**
- ✅ 일반 에러 바운더리 (error.tsx) 동작 확인
- ✅ "다시 시도" 버튼으로 에러 복구 확인
- ✅ "홈으로 돌아가기" 버튼으로 네비게이션 확인
- ✅ 에러 메시지 및 UI 표시 확인

### 2. 수동 테스트

#### 일반 에러 바운더리 테스트 (error.tsx)

임시로 컴포넌트에 에러를 발생시켜 테스트:

```tsx
// 예: dashboard/page.tsx에 임시로 추가
export default function DashboardPage() {
  throw new Error("테스트 에러");
  // ... 나머지 코드
}
```

#### 루트 레이아웃 에러 테스트 (global-error.tsx)

`app/layout.tsx`에서 직접 에러를 발생시켜 테스트:

```tsx
export default function RootLayout({ children }: { children: React.ReactNode }) {
  throw new Error("루트 레이아웃 에러 테스트");
  // ... 나머지 코드
}
```

### 3. 브라우저 개발자 도구 사용

콘솔에서 직접 에러를 발생시킬 수 있습니다:

```javascript
// React 컴포넌트 트리 밖에서 에러 발생
throw new Error("테스트 에러");
```

## 주의사항

⚠️ **프로덕션 환경에서는 `/test-error` 페이지를 제거하거나 접근을 제한해야 합니다.**

## 에러 바운더리 동작 원리

1. **일반 에러 바운더리 (`error.tsx`)**
   - `app` 디렉토리 내의 모든 페이지와 컴포넌트에서 발생하는 에러를 잡습니다
   - 레이아웃은 유지되며 에러가 발생한 부분만 대체됩니다

2. **전역 에러 바운더리 (`global-error.tsx`)**
   - 루트 레이아웃(`app/layout.tsx`)에서 발생하는 에러를 잡습니다
   - `html`과 `body` 태그를 포함한 완전한 HTML을 반환해야 합니다
   - 일반 에러 바운더리보다 우선순위가 높습니다

## 에러 로깅

에러는 `useEffect` 내에서 `console.error`로 로깅됩니다.
프로덕션 환경에서는 에러 리포팅 서비스(Sentry, LogRocket 등)에 전송하도록 확장할 수 있습니다.

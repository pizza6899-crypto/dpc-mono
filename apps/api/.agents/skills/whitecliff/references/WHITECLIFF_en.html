1.5 Flow of Data and Cronjob apimanual.general.1-5-title

In WHITECLIFF, every Debit will be paired with a Credit. The linkage is done through a matching transaction ID. All Debit without a Credit will be considered as pending. Integrators will need to prepare a Cronjob which monitors pending Debits and uses the Bet Results API to settle it within 10 minutes of Debit call time.

apimanual.general.1-5-1





Resend Credit & Bonus Mechanism

  • For any pending or error response Transaction ID, the Credit/Bonus Request API will trigger the Resend Mechanism.
  • The API automatically retries the request at 1 min, 5 min, and 10 min intervals if no response or error response is received from the agent's endpoint.
  • The auto retry process stops after 10 min.
  • After that, a manual resend can be triggered upon request.
  • Agents must implement proper handling for duplicate responses:
  • Credit API → "DUPLICATE_CREDIT"
  • Bonus API → "DUPLICATE_BONUS"
apimanual.general.1-5-2

Frequently Asked Questions Frequently Asked Question

Question

apimanual.question

Answer

apimanual.answer

What the differences between the "type"=0 and "type"=1 values in the Bonus Request API?

apimanual.faq.question-1

"type"=0 refers to an in-game bonus, which is considered a regular winning.

"type"=1 refers to a promotional bonus received from a provider's promotional event.

apimanual.faq.answer-1

Is an In Game Bonus, represented by "type"=0, will send through the Credit endpoint?

apimanual.faq.question-2

"type"=0 bonus are sent to the Bonus endpoint and will not be sent to the Credit endpoint. As a result, no Debit request will be generated for the integrator upon spin.

apimanual.faq.answer-2

What is the "credit_amount" parameter used for in the DEBIT Request API?

apimanual.faq.question-3

The "credit_amount" parameter represents the payout amount generated during a Debit request. If this parameter is present, integrators are required to process a credit settlement.


Note that this parameter applies only to the following products:

Habanero, BNG, PGsoft, Joker, Booming Games, Alize, JiLi, TaDa Gaming

apimanual.faq.answer-3

What causes the "Evolution: Currency Restriction" error?

apimanual.faq.question-4

Due to Evolution's policy, a player's currency must match their country of IP address. For example, players using IDR (Indonesian Rupiah) must access Evolution from Indonesia.

apimanual.faq.answer-4

What is the "type" parameter used for in the AUTH Request API?

apimanual.faq.question-5

The type parameter is used to launch Slots or Mini Games.

Integrators should set the type value using the game_id obtained from the Game List API.

apimanual.faq.answer-5

How to launch Slots or Mini Games?

apimanual.faq.question-6

To launch Slots or Mini Games, the integrator must use the "game_id" value from the Product Game List API and set it in the type parameter of the AUTH API.

apimanual.faq.answer-6

Dream Gaming

apimanual.faq.question-7

Dream Gaming "round_id", "game_id", and "table_id" are always 0 in Debit & Credit API requests. If you need the Game ID, please call the "Bet Results" API to update the game ID.

apimanual.faq.answer-7
